## 📊 Key Metrics

### **Critical Values to Monitor**

| Metric | Good | Acceptable | Bad |
|--------|------|-----------|-----|
| **ML Accuracy** | 48-52% | 45-55% | <40% or >60% |
| **Data Points** | 100+ | 50+ | <20 |
| **AI APIs** | 4/4 | 3/4 | <2/4 |
| **CPU Usage** | <30% | <50% | >80% |
| **Memory (MB)** | <400 | <600 | >1000 |
| **Std Dev** | <50% mean | <100% mean | >200% mean |

### **Interpretation Guide**

```
ML Accuracy 50% = ✅ Perfect (proves randomness!)
ML Accuracy 60% = ⚠️ Possible pattern or overfitting
ML Accuracy <45% = ❌ Model failure, retrain needed

Data Quality = Std Dev / Mean
  0-0.5 = Low variance (too predictable?)
  0.5-1.0 = Normal (good randomness)
  1-2 = High variance (very chaotic)
  >2 = Extreme (possible data issues)
```

## 📚 Reference Formulas

### **Kelly Criterion**
```
Optimal Bet = (bp - q) / b
  b = odds above 1 (e.g., 2.0x = b of 1)
  p = probability of win
  q = probability of loss (1 - p)
```

### **Risk of Ruin**
```
RoR = (q/p)^n
  Where you survive if RoR < 1
  Typical safety threshold: RoR < 0.01 (1%)
```

### **Provably Fair**
```
hash = HMAC-SHA256(server_seed, client_seed:nonce)
h = int(hash[:13], 16)
e = 2^52
crash = (100*e - h) / (e - h) / 100
if h % 33 == 0: crash = 1.0 (house wins)
```

### **Chi-Squared Test**
```
χ² = Σ(Observed - Expected)² / Expected
  DF = N - 1
  p > 0.05 = Data appears random ✅
```
